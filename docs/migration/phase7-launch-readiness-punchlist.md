# Phase 7 — launch-readiness punch-list

Living punch-list from the launch-readiness audit (issue [#102], epic [#67]).
Generated by a 35-page content/media/SEO diff against the live Webflow source
(`mapsnational.webflow.io`), each finding adversarially re-verified before
landing here. Nothing gets fixed off this list without a decision recorded
next to it — see the phase6 punch-list (`phase6-review-punchlist.md`) for the
prior round's format this one follows.

**Status:** `[ ]` open · `[x]` done · `[~]` in progress
**Type:** AFK (mechanical) · HITL (needs a decision)

## Coverage

- Content/media/SEO: all 35 assembled pages audited via the Payload API vs.
  the live Webflow page (or marked `no_live_equivalent` for net-new pages).
  One candidate finding (home page "missing Upcoming Events section") was
  rejected on adversarial re-check — it was category badges on existing
  cards, not a missing section.
- Media: **clean** — zero Webflow CDN references (`webflow.io` /
  `website-files.com` / `webflowusercontent.com`) found in any resolved media
  URL across all 35 pages. Satisfies the AC #2 "no rendered asset URLs point
  at the Webflow CDN" criterion outright.
- Responsive: **sampled, not exhaustive.** Phase 6 already did a full visual
  pass; this round manually spot-checked mobile/tablet/desktop on the areas
  touched by this week's commits (post-gallery slider, mobile hamburger nav,
  footer CTA, the private-sector-engagement slug move) and found nothing
  broken. The other ~30 pages were not re-screenshotted at every breakpoint.
- Gated `/members/*` pages: content was audited via the public Payload API
  (bypasses the Outseta route gate), but a real logged-in visual/interactive
  pass was **not** done — needs a human with an Outseta account.

## High — content gaps needing editorial input before fixing

- [x] **LR2 — `/about-us/partners` missing 3 logos + inconsistent sizing/contrast** _(done)_ → re-hosted Sahara, De Lune Corp, and Myna from the live CDN via `rehost:images` (`scripts/rehost-images.ts` REMOTE_LOGOS) and wired into `LOGO_FILENAMES` (`scripts/seed-pages.ts`), same pattern as the prior punch-list's [P1]. Grid now shows 34 logos (was 31). Also fixed two related visual bugs found while doing this, both in [LogoStrip/Component.tsx](../../src/blocks/LogoStrip/Component.tsx): (1) logos were constrained to a fixed **height** only (`w-auto`), so a wide wordmark and a square/tall mark rendered at very different visual sizes — changed to a fixed **box** (`h-12 w-28 md:h-14 md:w-32`) with `fill` + `object-contain` so every logo occupies the same footprint regardless of source aspect ratio; (2) real partner logos are a grab-bag of raw exports (some carry their own white background rectangle, some are dark-on-transparent, e.g. MLSA), which read badly floating directly on the dark theme's near-black page — added a fixed white card (`bg-[var(--neutral-white)]`, rounded, bordered, shadowed) behind every logo regardless of theme, the same theme-invariant-white pattern `PostHero` already uses for its preview card. Verified at desktop + mobile, both themes: all 34 logos legible and uniformly sized. A third issue surfaced once boxes were pixel-equal: several source files carry their own baked-in whitespace/transparent padding in wildly different amounts (e.g. ISPU's canvas was ~35% padding), so same-size boxes still read as different visual "ink" sizes. Fixed with a one-off script ([trim-logo-padding.ts](../../scripts/trim-logo-padding.ts)) that runs every partner logo through `sharp().trim()` (already a project dependency) and re-uploads the trimmed result into the same Media doc id — 29 of 34 files were meaningfully cropped. Re-verified at desktop, both themes: logos now carry consistent visual weight, no more oversized margins on some cards vs. tight crops on others. _(AFK — precedent via `rehost:images`; all three fixes were mechanical, no design judgment calls)_
